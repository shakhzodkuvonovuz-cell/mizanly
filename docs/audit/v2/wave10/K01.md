## Summary

**The CI pipeline covers approximately 40% of the test surface.** It runs API unit tests, API integration tests, API build, API+mobile typecheck, and Go e2e-server tests. It completely misses:

- **livekit-server Go tests** (123 tests, the entire calling backend)
- **Signal Protocol tests** (633 tests, the entire E2E encryption stack)
- **LiveKit/CallKit mobile tests** (49 tests, call-related utilities)
- **ESLint** (rules exist but are never run)
- **Prettier** (script exists but is never run)
- **Coverage** (no thresholds, no collection)
- **Docker builds** (2 Dockerfiles never validated)
- **Security scanning** (no Dependabot, no npm audit)
- **shared package typecheck**
- **workers/exif-stripper build**

The Go version mismatch (1.25 in code vs 1.26 in livekit Dockerfile) and empty ESLint configs (rules: {}) mean that even the existing checks are weaker than they appear.
