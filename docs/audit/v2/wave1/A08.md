## Cross-Module Concerns

1. **Inconsistent internal auth patterns.** `internal-push.controller.ts` uses raw string comparison (`!==`) for service key auth, while `internal-e2e.controller.ts` (in the messages module) uses HMAC-SHA256 + `timingSafeEqual`. These two controllers serve the same purpose (Go microservice -> NestJS) but have different security postures.

2. **Webhook dispatch is synchronous in request path.** `webhooksService.dispatch()` performs up to 100 webhook deliveries with 3 retries each (up to 7 seconds per delivery with backoff). If called from a request handler, it could block for minutes. The caller should use the queue service or fire-and-forget.

3. **Notification creation has no limit on concurrent creates.** The `NotificationsService.create()` method is called by many modules. A viral post could trigger thousands of simultaneous notification creates (one per liker), all hitting Prisma + Redis in parallel. No backpressure mechanism exists.
