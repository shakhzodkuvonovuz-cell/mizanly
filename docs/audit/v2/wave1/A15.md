## Critical Finding Detail

### Finding #1: Plaintext PIN overwrites hashed PIN (CRITICAL)

**File:** `apps/api/src/modules/parental-controls/parental-controls.service.ts`, lines 180-183

```typescript
// Line 180-183 in updateControls():
return this.prisma.parentalControl.update({
  where: { id: control.id },
  data: dto,  // <-- dto includes { pin: "123456", restrictedMode: true, ... }
});
```

The `UpdateParentalControlDto` has a required `pin: string` field (the plaintext PIN for verification). But the entire DTO is spread into the Prisma `data` parameter. Since the Prisma `ParentalControl` model also has a `pin: String` column (which stores scrypt hashes like `"a1b2c3...:d4e5f6..."`), Prisma happily writes `"123456"` into the `pin` column, overwriting the hash.

**Reproduction:**
1. Parent links child with PIN `123456` -> DB stores `"salt:scryptHash"`
2. Parent updates restrictions with PIN `123456` -> DB stores `"123456"` (plaintext!)
3. Parent tries to verify PIN `123456` -> `verifyPin()` calls `"123456".split(':')` -> gets `["123456"]` -> `salt = "123456"`, `hash = undefined` -> `Buffer.from(undefined, 'hex')` throws TypeError

**Fix:** Destructure `pin` out before passing to Prisma:
```typescript
const { pin: _verifiedPin, ...updateData } = dto;
return this.prisma.parentalControl.update({
  where: { id: control.id },
  data: updateData,
});
```
