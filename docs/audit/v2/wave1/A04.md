## Cross-Module Issues

1. **Threads `getById` / `getThreadUnroll` / `shareToStory` all ignore the `visibility` field.** The `ThreadVisibility` enum supports `PUBLIC`, `FOLLOWERS`, and `CIRCLE`, and the `CreateThreadDto` validates it on creation, but no read endpoint enforces it. This means the visibility feature is effectively decorative — all threads are public to anyone with the ID.

2. **Content moderation gap:** `create` is the only mutation that calls `contentSafety.moderateText()`. The three other content-writing paths (`updateThread`, `createContinuation`, `addReply`) skip moderation entirely. This creates a trivial bypass: create a safe thread, then edit it to contain blocked content.

3. **Majlis Lists timeline does not apply the same user-state and scheduling filters that the Threads feed uses.** The threads feed correctly filters `isBanned`, `isDeactivated`, `isDeleted`, and `scheduledAt`, but the majlis-lists timeline does none of these.
