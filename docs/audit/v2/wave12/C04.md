### Most Critical Finding

**Finding #1 (C):** The LiveKit API service (`livekit.ts`) constructs URLs by passing full absolute URLs to `api.post()/api.get()`, which prepends `API_URL` to them. This means every LiveKit call produces a URL like `http://localhost:3000/api/v1https://livekit.mizanly.app/api/v1/calls/rooms` -- completely broken. All calling functionality (the primary real-time feature) is non-functional. This was never caught because no real-device testing has occurred.
