## Summary

**Severity Counts:** C:0 H:3 M:12 L:9 I:4 **Total: 28**

### Critical Patterns Found

1. **Stale closure epidemic in `useLiveKitCall`** (findings 1-3): Multiple callbacks capture React state values via closure instead of refs. The `toggleMute`/`toggleCamera`/`toggleSpeaker`/`toggleScreenShare` family all share the same bug pattern: `useCallback` depends on state value, rapid invocations read stale state. The fix is consistent: use refs for values read inside callbacks, or use functional state updates.

2. **E2EE salt fallback silently breaks encryption** (finding 4): The `Date.now()` fallback for missing `e2eeSalt` means two clients derive different ratchet salts. This is worse than crashing — it silently produces garbled audio/video that looks like "the call connected but I can't hear anything." Should throw an error instead of falling back.

3. **Async effects without cancellation guards** (findings 7, 9): `usePushNotificationHandler` and `useOfflineFallback` both start async operations inside useEffect but don't properly guard against the async operation completing after unmount/re-subscription.

4. **Non-reactive ref returns** (findings 6, 12): `sessionId` and `preloadCount` are read from refs during render, making them invisible to React's reactivity system. Changes to these values never trigger re-renders.

### What's Good

- `useLiveKitCall` has thorough E2EE key zeroing (F5 fix), proper room cleanup on unmount, and the `mountedRef` pattern prevents setState-after-unmount.
- `useContextualHaptic` is properly memoized with `useMemo(() => ..., [])`.
- `useScrollLinkedHeader` correctly uses Reanimated shared values and worklets.
- `useResponsive` properly cleans up its Dimensions listener.
- `useNetworkStatus` is minimal and correct.
- `useChatLock` uses `SecureStore` (hardware-backed) for sensitive data.
- `useAmbientColor` has proper async cancellation with the `cancelled` flag.
- All hooks have proper TypeScript types (no `any` in non-test code).
