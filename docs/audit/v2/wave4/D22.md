### Critical Findings (H severity):

1. **#20-21 (majlis-list/[id].tsx):** Container has no tc override for background (permanently dark) AND no safe area handling (content renders behind header). This screen is visually broken.
2. **#25 (majlis-list/[id].tsx):** No error state. API failure shows empty state instead of error + retry.
3. **#38 (majlis-lists.tsx):** Delete mutation has no per-item loading state, allowing duplicate mutations.
4. **#50 (manage-broadcast.tsx):** No query invalidation after promote/demote/remove mutations. Stale data persists.
5. **#51 (manage-broadcast.tsx):** Screen fetches channel owner's FOLLOWERS instead of actual broadcast subscribers. The entire screen shows wrong data by design (comment admits "simulate the list").
