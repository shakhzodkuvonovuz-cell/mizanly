### Critical patterns across all 5 screens:

1. **Light mode broken everywhere (H)**: All 5 screens use `colors.text.*` instead of `tc.text.*` for text colors. This is a systemic issue -- in light mode, white text on white backgrounds makes every screen unreadable.

2. **No RTL support on 3 of 5 screens**: `product-detail.tsx`, `qibla-compass.tsx`, and `qr-code.tsx` have zero RTL handling. Given 3 of 8 supported languages (Arabic, Urdu, possibly Malay) are RTL, this breaks layout for a significant user segment.

3. **fontWeight without fontFamily**: `qibla-compass.tsx`, `qr-code.tsx`, and `profile/[username].tsx` use raw `fontWeight` values without corresponding `fontFamily` from the `fonts.*` system. On Android, this causes incorrect font face selection.

4. **Dimensions.get at module scope**: `product-detail.tsx`, `qibla-compass.tsx`, and `profile/[username].tsx` capture window dimensions at import time, making them stale on rotation/split-view.
