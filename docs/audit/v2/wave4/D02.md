### Critical Patterns Across All 5 Screens

1. **`colors.dark.*` / `colors.text.*` hardcoded in StyleSheet** (38 findings): Every screen hardcodes dark-theme colors in `StyleSheet.create()`. Most are overridden inline with `tc.*`, but `appeal-moderation.tsx` L683 `detailsInput.color` is NOT overridden inline, meaning input text is invisible in light theme.

2. **No `useContextualHaptic` on 3/5 screens**: `analytics.tsx`, `appeal-moderation.tsx`, and `archive.tsx` have zero haptic feedback on any interaction, violating the mandatory haptic rules.

3. **Missing error handlers on mutations**: `ai-assistant.tsx` (2 mutations), `archive.tsx` (2 mutations) silently swallow API errors with no `onError` callback and no `showToast`.

4. **`Dimensions.get('window')` at module scope**: Used in `analytics.tsx`, `appeal-moderation.tsx` (dead code), and `archive.tsx`. The value is stale on orientation change or multitasking.

5. **No `KeyboardAvoidingView`**: Both `ai-assistant.tsx` and `appeal-moderation.tsx` have TextInputs inside ScrollViews with no keyboard avoidance.

6. **Hardcoded content preview**: `appeal-moderation.tsx` L177 shows a fake English string instead of the actual moderation-flagged content from the API.

7. **Deprecated `Clipboard` from react-native**: `ai-assistant.tsx` uses the deprecated `Clipboard` API instead of `expo-clipboard`.
