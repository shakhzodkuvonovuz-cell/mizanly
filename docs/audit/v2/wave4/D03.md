### Critical Path Issues

1. **H #15 + H #16 (audio-room.tsx):** "End Room" and "Accept/Decline Hand" buttons visible to all participants, not just hosts. If the API doesn't guard these actions server-side, any listener can destroy a room or promote themselves to speaker.

2. **H #7 (audio-library.tsx):** Favorites feature is completely non-functional. `isFavorite` is hardcoded false, toggle does nothing server-side. The entire favorites filter is decorative.

3. **M #30 (banned.tsx):** `signOut()` has no error handling -- unhandled promise rejection if Clerk call fails.

4. **M #36 (biometric-lock.tsx):** `authenticateAsync` has no error handling -- hardware failures crash the app.

5. **M #9 (audio-library.tsx):** Double-tap on play can create two simultaneous audio instances.

### Pattern Issues Across All 5 Screens

- **Hardcoded dark theme in StyleSheet:** All 5 screens use `colors.dark.*` / `colors.text.*` in their `StyleSheet.create()` definitions. While most override inline with `tc.*`, the base styles are dark-only. Any missed inline override breaks light theme.
- **No visual press states:** No screen uses `Pressable` style-as-function for press feedback.
- **Missing disabled states during async:** Buttons that trigger API calls lack `disabled` during pending state, enabling double-tap issues.
