### Critical Patterns Across All 5 Screens

1. **Light mode universally broken (A1):** Every screen hardcodes `colors.dark.*` and `colors.text.*` in `StyleSheet.create()` instead of using `tc.*` from `useThemeColors()`. Inline JSX sometimes uses `tc.*` correctly but stylesheet styles override them. This is the dominant finding — ~45 of 80 findings are A1 violations.

2. **No keyboard handling (B19):** 4 of 5 screens have TextInputs inside ScrollViews without `keyboardShouldPersistTaps="handled"` or `KeyboardAvoidingView`. Only `create-thread.tsx` has `keyboardShouldPersistTaps`.

3. **No double-tap protection (B13):** All 4 create screens call `mutation.mutate()` directly from button press with no debounce guard. A rapid double-tap before `isPending` state propagates will fire the mutation twice.

4. **No offline awareness (C29):** All 4 create screens upload media blobs without checking network status first. Videos can be hundreds of MB.

5. **Silent error swallowing (C23):** 8 instances of empty `catch` blocks or `// ignore` comments in draft save/load paths.

6. **Raw Image in dashboard (E37):** `creator-dashboard.tsx` imports and uses `Image` from `react-native` instead of `ProgressiveImage` for post thumbnails.
