### Cross-cutting themes

1. **Light theme not supported (A1):** All 5 screens hardcode `colors.dark.*` and `colors.text.*` in their StyleSheet definitions. While some have inline `tc.*` overrides, the coverage is inconsistent. If a light theme is ever activated, most screens will have dark backgrounds with dark text.

2. **RTL layout ignored (A2):** 4 of 5 screens import `isRTL` but never use RTL layout utilities (`rtlFlexRow`, `rtlTextAlign`) for their hardcoded `flexDirection: 'row'` styles. Leaderboard is the only screen that properly applies RTL throughout. The Islamic calendar is the most impactful omission since Arabic-speaking users would expect RTL day ordering.

3. **No haptic feedback (B10/E44):** islamic-calendar.tsx and live/[id].tsx have zero haptic feedback. local-boards.tsx has minimal haptic. Only leaderboard.tsx and link-child-account.tsx properly use `useContextualHaptic`.

4. **fontWeight instead of fontFamily (A7):** 4 of 5 screens use `fontWeight: '600'/'700'` instead of `fontFamily: fonts.bodySemiBold/fonts.bodyBold`. Only leaderboard.tsx consistently uses the font family tokens. This means custom fonts may not render correctly.

5. **No double-tap guards (B13):** Navigation Pressables in all 5 screens lack double-tap protection. Rapid tapping pushes duplicate screens.

6. **No keyboard handling (B19):** Screens with TextInput (link-child-account, live/[id], local-boards) lack `KeyboardAvoidingView`.
