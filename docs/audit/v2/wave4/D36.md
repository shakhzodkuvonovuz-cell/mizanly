### Top Issues by Category

**RTL (A2):** 3 screens (story-viewer, surah-browser, tafsir-viewer) have zero or broken RTL support. This is critical for an app targeting Arabic/Urdu-speaking Muslims -- 2 of the 8 supported languages are RTL. The Quran browser and tafsir viewer being broken for Arabic users is especially severe.

**Light Theme (A1):** 4 of 5 screens use `createStyles(tc)` but still reference `colors.text.*` instead of `tc.text.*`. If light theme is ever enabled, text will be invisible or unreadable.

**Error Handling (C23):** 4 of 5 screens have missing or misleading error states. Streaks and surah-browser show empty states on network errors. Story-viewer silently swallows API errors. Tafsir-viewer's error boundary doesn't wrap all render paths.

**Safe Area (A3):** 3 screens use hardcoded `paddingTop` values (100-110px) instead of calculating from `useSafeAreaInsets()`. Bottom safe area is missing on 3 screens.

**Typography (A7):** story-viewer and tafsir-viewer use `fontWeight` strings instead of `fonts.*` family tokens, bypassing the custom font system.

**Haptic (B10):** tafsir-viewer has zero haptic feedback on any interaction. story-viewer is inconsistent (reactions have haptic, navigation taps don't).
