### Cross-Cutting Issues (affect all 5 screens)

1. **Dark-mode-only StyleSheets**: All 5 screens use `colors.dark.*` and `colors.text.*` in `StyleSheet.create()` instead of `tc.*`. This is the single biggest issue — every screen is broken in light mode. Only JSX-inline styles use `tc.*` correctly; the static styles are all hardcoded dark.

2. **No double-tap protection**: All 5 screens rely solely on `isPending`/`submitting` state to prevent double-tap. React state updates are async — a fast double-tap fires the mutation/API call twice before the flag updates. Need a `useRef` guard or debounced press handler.

3. **No offline detection**: None of the 5 screens check network status before API calls. Users compose content, tap submit, and get an opaque error.

4. **Keyboard handling**: 3 of 5 screens lack `keyboardShouldPersistTaps="handled"`. The remaining 2 have it but lack `KeyboardAvoidingView` for bottom-positioned elements.
