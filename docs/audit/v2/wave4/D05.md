### Critical Issues (H)

1. **#30** `broadcast/[id].tsx` L343: Delete message has no confirmation dialog -- destructive action with no undo.
2. **#46** `call-history.tsx` L69: Rows with deleted/missing participants silently render null, causing invisible gaps.
3. **#88** `camera.tsx` L43-87: Camera screen does not use the actual camera at all. Capture button is a navigation shortcut. The entire `expo-camera` permission flow is pointless since `CameraView` is never rendered.

### Most Impactful Patterns

- **RTL breakage** (8 findings): `left:/right:` used instead of `start:/end:`, `marginRight` instead of `marginEnd`. Arabic and Urdu (2 of 8 supported languages) will have broken layouts.
- **Silent error swallowing** (7 findings): Catch blocks only log in `__DEV__`, no user-facing error feedback. Users get no indication of failures.
- **Missing haptics** (4 screens): broadcast-channels, broadcast/[id], camera all lack `useContextualHaptic()` entirely.
- **Static Dimensions** (2 findings): Module-level `Dimensions.get('window')` never updates on rotation/split-screen.
- **Theme-hardcoding** (20+ findings): Static StyleSheet uses `colors.dark.*` and `colors.text.*` instead of `tc.*` from `useThemeColors()`.
