## Summary

**C:3 H:10 M:16 L:8 I:6 Total: 43**

### Critical (3)
- saf.tsx: Inline `onViewableItemsChanged` breaks FlashList viewability tracking
- saf.tsx: `globalThis` dwell tracking leaks memory unboundedly
- bakra.tsx: Unstable `handleViewableItemsChanged` callback breaks video autoplay

### High (10)
- ALL: `colors.dark.*` and `colors.text.*` static tokens in StyleSheet = light mode broken
- bakra.tsx: `handleShare` has no try/catch = unhandled rejection
- bakra.tsx: Hardcoded `#1C1C1E` background color
- minbar.tsx: VideoCard text colors not theme-aware
- minbar.tsx: Silent error swallowing on watch-later save
- majlis.tsx: `trendingHeaderText` uses static dark token

### Top 5 fixes by impact
1. **Memoize `onViewableItemsChanged`** in saf.tsx (C) -- prevents silent breakage of dwell tracking
2. **Stabilize `handleViewableItemsChanged`** in bakra.tsx (C) -- prevents video autoplay breakage
3. **Replace `globalThis` dwell store** with a `useRef(new Map())` in saf.tsx (C) -- fixes memory leak
4. **Add tc.* inline overrides** to all text styles across all 4 screens (H) -- fixes light mode
5. **Add try/catch + toast** to `handleShare` and `handleSaveToWatchLater` (H) -- prevents crashes
