### Critical Findings (C)

1. **#16 parental-controls.tsx L419**: Destructive "Unlink Account" action has no confirmation dialog. PIN is auth, not intent confirmation.
2. **#24 parental-controls.tsx L504**: Parental control data is fetched BEFORE PIN verification via `hasControlsQuery` (no `enabled: pinVerified` guard). The PIN gate is cosmetic -- data is in React Query cache.
3. **#46 pinned-messages.tsx L88**: Unpin action fires immediately on tiny x-button tap with no confirmation. Affects all group chat members.

### Most Impactful Patterns

- **Light mode broken on ALL 5 screens**: Every screen uses static `colors.text.*` or `colors.dark.bg` instead of theme-aware `tc.text.*`. Playlist is worst (entire container hardcoded dark).
- **RTL broken on 4 of 5 screens**: Only `parental-controls.tsx` has RTL support. The other 4 have zero RTL handling.
- **No double-tap guards anywhere**: All 5 screens allow duplicate navigation/mutation on rapid taps.
- **React Hooks violation**: `parental-controls.tsx` line 607 calls `useEffect` after conditional returns, which will crash React.
