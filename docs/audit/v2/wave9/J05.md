# J05 — Bundle Size Audit (Mobile App)

**Scope:** `apps/mobile/package.json`, import patterns, asset optimization, duplicate libraries, missing dynamic imports

---

## Critical Findings

| # | Severity | Finding | File(s) | Impact | Recommendation |
|---|----------|---------|---------|--------|----------------|
| 1 | **CRITICAL** | `@tensorflow/tfjs` + `@tensorflow/tfjs-react-native` + `nsfwjs` in production dependencies (~4-8MB native + ~2MB JS) | `package.json` lines 37-38, 72 | These three packages are never statically imported anywhere in the codebase. `nsfwCheck.ts` uses `require()` dynamic imports which Metro still bundles. TF.js alone adds ~2MB minified JS + native binaries. The NSFW model file (`model.json` + shard) isn't even bundled in assets yet. | Move to `devDependencies` or remove entirely. The `nsfwCheck.ts` service already gracefully degrades (`{ safe: true, checked: false }`) when packages are missing. Server-side moderation is the real gate. If keeping client-side screening, load via `expo-modules` lazy native module or OTA download — do NOT ship TF.js in the main bundle. |
| 2 | **CRITICAL** | `ffmpeg-kit-react-native` in production dependencies (~15-30MB native binary for full-gpl variant) | `package.json` line 67 | FFmpeg-kit full-gpl is one of the largest React Native native dependencies. It's used by exactly 2 files: `ffmpegEngine.ts` and `create-reel.tsx`. The video editor is a niche feature — most users will never touch it. The entire FFmpeg native binary ships to every user regardless. | This cannot be tree-shaken since it's a native module. Consider: (a) using `min` or `min-gpl` variant instead of `full-gpl` to halve binary size, (b) documenting that this is a known ~15-30MB cost, or (c) deferring video editing to a future "pro" download. |
| 3 | **CRITICAL** | All 8 i18n JSON files (1.2MB total) statically imported and bundled at startup | `src/i18n/index.ts` lines 7-14 | Every language file is synchronously imported: `import en from './en.json'` through `import ms from './ms.json'`. At 1.2MB raw (146-176KB each), this adds ~1.2MB to the JS bundle parsed at cold start. A Turkish user still loads Arabic, Bengali, Malay, etc. | Load only the detected language synchronously (`en` + user's language). Lazy-load remaining languages on language switch via `i18next.addResourceBundle()`. This cuts initial bundle parse by ~1MB for most users. |

## Summary

### Estimated Bundle Impact

| Category | Estimated Savings | Effort |
|----------|------------------|--------|
| Remove TF.js/nsfwjs from prod deps (#1) | ~2-4MB JS + native | 10 min |
| Remove 4 unused native packages (#4-7) | ~500KB-1MB native | 10 min |
| Move 5 build-time deps to devDependencies (#9-13, #24) | 0 runtime (cleaner deps) | 10 min |
| Lazy-load i18n languages (#3) | ~1MB JS parse time | 2 hours |
| FFmpeg-kit variant downgrade (#2) | ~10-15MB native | 1 hour |
| Remove/verify GIPHY SDK (#8) | ~5MB native if unused | 30 min |
| **Total potential savings** | **~19-26MB** | **~4 hours** |

### What's Done Right

- `@noble/*` uses subpath imports (minimal crypto bundle)
- `date-fns` uses named exports (tree-shakeable)
- `lucide-react-native` uses named icon imports (not barrel)
- `lazily()` utility exists (just not used yet)
- NSFW check uses dynamic `require()` with graceful degradation
- Expo Router provides route-level code splitting
- Sound files are small placeholders (not bloated WAVs)
- Asset images are reasonably sized (~20KB PNGs)
