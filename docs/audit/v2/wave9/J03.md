The codebase has several systemic performance anti-patterns. The main tab screens (saf, bakra, majlis, minbar, risalah) generally follow good practices (memoized renderItem, useCallback, memo on list items), but secondary screens, list components, and shared patterns across 55+ files have significant issues that will cause unnecessary re-renders at scale.

## Systemic Patterns

### Pattern 1: `createStyles(tc)` in component bodies (149 occurrences, 73 files)

This is the single most impactful issue. The pattern:
```tsx
function SomeComponent() {
  const tc = useThemeColors();
  const styles = createStyles(tc); // <-- NEW StyleSheet every render
  ...
}
```

appears in virtually every themed component. When used in list items, it multiplies: 50 visible items x render frequency = hundreds of wasted StyleSheet.create() calls.

**Recommended solution:**
1. For screen-level components: `const styles = useMemo(() => createStyles(tc), [tc])`
2. For list item components: compute styles once in parent, pass via context or props
3. Long-term: migrate to a cached theme-styles hook that returns a stable reference when theme hasn't changed

### Pattern 2: Inline callbacks in FlatList props (77 renderItem + 36 ListEmptyComponent + 17 ListFooterComponent = 130 inline functions)

The 5 main tab screens follow best practices. The 55+ secondary screens do not. This is a consistency gap.

### Pattern 3: Unmemoized derived arrays from React Query

Every screen that uses `useInfiniteQuery` computes `const items = query.data?.pages.flatMap(...) ?? []` without `useMemo`. These run on every render, even when the underlying query data hasn't changed.

---

## Priority Fix Order

1. **`createStyles(tc)` in list item components** (C1) â€” highest frequency, easiest fix, biggest payoff
2. **Inline `renderItem` on heavy list screens** (C2) â€” 77 occurrences, each one defeats FlatList optimization
3. **`NotificationRow` not memoized** (M6) â€” notifications is a high-traffic screen with 50+ items
4. **`filtered` array in risalah** (M1) â€” chat list re-renders on every typing indicator / online status change
5. **Unmemoized `flatMap` derived arrays** (M2, m4, m7) â€” low effort, moderate payoff across all feeds
