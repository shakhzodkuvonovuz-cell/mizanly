## TOP 5 FIXES BY IMPACT

1. **Add RTL utilities to the 55 screens** — All 55 screens need `rtlFlexRow(isRTL)` on every `flexDirection: 'row'` container and `rtlTextAlign(isRTL)` on key text elements. This is a systemic issue affecting 3 of 8 supported languages (Arabic, Urdu, partial Malay). **Highest priority: prayer-times, zakat-calculator, schedule-post, stitch-create** (15+ row layouts each).

2. **Replace all hardcoded English a11y hints in bakra.tsx tab** — 5 hardcoded English `accessibilityHint` strings. Since bakra is a main tab, every user with VoiceOver/TalkBack hears English regardless of language.

3. **Translate prayer names** — `PRAYER_NAMES` at `prayer-times.tsx:35` must use `t('islamic.fajr')` etc. instead of hardcoded English strings. Islamic screens are core to the app's identity.

4. **Replace `borderLeftWidth`/`borderLeftColor` with `borderStartWidth`/`borderStartColor`** — 9 screens have decorative accent bars using LTR-only border properties. These render on the wrong side for RTL users.

5. **Fix hardcoded USD currency** — `gift-shop.tsx`, `cashout.tsx`, `creator-dashboard.tsx`, `zakat-calculator.tsx` all hardcode `$` symbol. Use `Intl.NumberFormat` with user's locale or at minimum make the currency symbol configurable.
