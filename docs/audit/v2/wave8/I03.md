## Analysis by Check

### Check 1 — Hardcoded English strings (28 findings)
The most severe cluster is `community-guidelines.tsx` where the entire GUIDELINES data structure (5 sections, 20 rules) is hardcoded English with zero i18n. Other significant clusters:
- `create-story.tsx` FILTERS and FONTS arrays (11 labels)
- `creator-dashboard.tsx` TABS array (4 labels, though translated copies exist downstream)
- `create-video.tsx` CATEGORIES array (11 constants used as display labels)
- `conversation/[id].tsx` encrypted message error strings, voice player a11y labels
- `create-event.tsx` `toLocaleString('en-US')` forces English date format

### Check 2 — Missing i18n keys (1 finding)
Most screens use `t()` correctly. One key (`messages.tapToViewProfile`) uses a fallback string suggesting it may not be defined in all 8 locale files.

### Check 3 — RTL violations (7 findings)
All concentrated in `conversation/[id].tsx` with 7 instances of `marginLeft` in both inline styles and StyleSheet. The codebase generally uses `marginStart`/`marginEnd` correctly elsewhere, but conversation screen has legacy `marginLeft` usages that break ar/ur layout.

### Check 4 — Missing a11y roles (14 findings across checks 4+5)
`conversation-info.tsx` is the worst offender with 10 action row Pressables (starred messages, pinned, media, export, disappearing, wallpaper, theme, lock, encryption, leave) that all lack `accessibilityRole` and `accessibilityLabel`.

### Check 5 — Missing a11y labels on icon-only buttons (14 findings)
Spread across multiple screens. Icon-only buttons (close, back, dismiss, time-adjust) lack screen reader descriptions.

### Check 6 — Screen reader order
No issues found. All screens use logical component ordering that matches visual layout. FlatLists and ScrollViews maintain natural reading order.
