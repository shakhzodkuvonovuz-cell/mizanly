# S01 -- Prisma Schema Audit (Lines 1-2500)

**Scope:** Every model, field, enum, index, and relation in `apps/api/prisma/schema.prisma` lines 1-2500.

**Audit criteria:** Money types, timestamp types, defaults, table names, enum completeness/usage, optional vs required, `@updatedAt`, String-should-be-enum, redundant indexes, missing FK indexes, `onDelete` correctness, plaintext credentials, missing unique constraints, naming consistency.

**Exclusions:** Findings already reported in wave2 (B01-B12) are not re-reported. This audit focuses exclusively on schema-level structural issues.

| 1 | C | schema.prisma:1670 | **`Conversation.lockCode` stored as plaintext `String?`.** This is a secret code used to lock/unlock conversations. Storing it in plaintext means any DB read (admin panel, SQL injection, DB backup leak) exposes all conversation lock codes. Should be stored as a bcrypt/argon2 hash. | Plaintext credential storage. Any DB breach exposes all conversation lock codes, allowing unauthorized access to locked conversations. | S -- hash on write, compare on read |
| 2 | C | schema.prisma:2435 | **`CallSession.e2eeKey` stored as `Bytes?` in the DB with no encryption at rest.** The 32-byte E2EE key for SFrame call encryption is stored in plaintext binary. CLAUDE.md acknowledges this ("Server briefly holds key material, wiped on call end"), but if the wipe fails (server crash, missed webhook), the key persists indefinitely in the DB. Combined with the ratchet key in `Message.e2eSenderRatchetKey`, a DB breach exposes both call and message crypto material. | E2EE key material at rest in plaintext. DB breach + retained keys = full call decryption. | M -- encrypt at rest with a server-side KMS key, or ensure guaranteed wipe via DB trigger/TTL |

## Summary

**Severity counts:** C:2 H:6 M:16 L:12 I:6 **Total: 42**

### Critical (2)
- **#1** Conversation lock code stored as plaintext String (should be hashed)
- **#2** E2EE call key stored as plaintext Bytes with no guaranteed wipe

### High (6)
- **#3** `Channel.totalViews` uses `Int` (overflows at 2.1B)
- **#4** `ConversationMember.role` is freetext String (already caused bugs per B06)
- **#5** `CallParticipant.role` is freetext String (no constraint)
- **#6** `Tip.status` is freetext String despite `TipStatus` enum existing
- **#7** `Story` model missing `isRemoved` for soft-delete/moderation
- **#8** `UserSettings` permission fields are freetext Strings (privacy risk)

### Patterns identified
1. **12 models missing `@updatedAt`** on mutable records (#9, 13, 20-27, 36, 38, 39)
2. **6 redundant indexes** where `@unique` already creates an index (#15-19, 18)
3. **7 String fields that should be enums** (#4-6, 8, 32-34, 44)
4. **3 models missing `isRemoved`** for proper soft-delete (#7, 10, 11)
