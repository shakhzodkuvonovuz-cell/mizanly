# S02 -- Prisma Schema Lines 2501-5037 (Deep Audit)

**Scope:** Every model, field, enum, relation, index, and default value in `apps/api/prisma/schema.prisma` from line 2501 to end of file (line 5037). ~100 models, ~1,000 fields.

**Method:** Read full schema section. Cross-referenced every model against `apps/api/src/modules/**/*.service.ts` for usage. Checked every enum for at least one model-field consumer. Verified index coverage, `@updatedAt` presence on mutable models, type correctness (Decimal for money, DateTime for timestamps, enums for constrained strings), credential storage, `onDelete` semantics, and relation completeness.

**Excluded:** Findings already reported in Wave 2 (B01-B12) are not re-reported.

## Summary

**Severity counts:** C:4 H:12 M:27 L:16 I:7 Total: 66

(Finding #57 and #62 are documentation/already-reported and excluded from counts.)

### Critical (4)
- 2x plaintext/weakly-hashed credential storage (TOTP secrets, backup codes)
- 2x plaintext webhook credentials (secret + token)

### High (12)
- 8x String fields that should be enums (Tip.status, AudioRoom.status, AudioRoomParticipant.role, CharityDonation.status, ZakatFund.status, PlaylistCollaborator.role, MosqueMembership.role, MosqueCommunity.madhab)
- 4x dead/orphaned models with no service

### Medium (27)
- 15x mutable models missing `@updatedAt` (financial models like Tip, PostPromotion, MembershipSubscription are most concerning)
- 6x String fields that should be enums (UserSettings permissions, EventRSVP.status, PostPromotion.status, ParentalControl fields)
- 3x String fields that should be Json type (openingHours, checklistJson, criteria)
- 2x dangling FK references (Embedding.postId/userId, ChatFolder.conversationIds)
- 1x orphaned model with no creation API (CreatorEarning)

### Low (16)
- 7x more String-should-be-enum cases (GeneratedSticker.style, MembershipTier.level, etc.)
- 4x missing referential integrity on polymorphic/dangling FKs
- 2x missing counter reconciliation
- 1x dead enum (TipStatus)
- 1x String-should-be-Json
- 1x orphaned analytics model

### Info (7)
- 7x redundant indexes (standalone index on column already covered by unique/PK leading column)
