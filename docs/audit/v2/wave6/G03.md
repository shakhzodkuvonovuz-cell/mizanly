## Summary of Critical Gaps

The e2e-server infrastructure has several meaningful gaps compared to the livekit-server (built 1 day later and incorporating lessons learned):

1. **No CORS middleware** -- livekit-server has it, e2e-server does not
2. **No config package** -- env vars scattered across 3 files with inconsistent validation
3. **No request ID middleware** -- no way to correlate log lines from the same request
4. **Rate limit bypass on empty params** -- silently skips rate limiting instead of rejecting
5. **`http.DefaultClient` for webhooks** -- follows redirects (SSRF risk), no TLS pinning
6. **Auth middleware untested** -- the most security-critical code has 0% test coverage

None of these are exploitable remotely today (Clerk JWT validation is solid, rate limiting fails closed on Redis errors), but they represent defense-in-depth gaps that the livekit-server has already addressed.
