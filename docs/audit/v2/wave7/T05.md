## Module-by-Module Summary

### videos (service + controller)
- **Service:** 24 public methods. 19 have at least one test. 5 completely untested: `getRecommended`, `deleteComment`, `incrementPremiereViewerCount`, `invalidateVideoFeedCache` (private but called from public methods), and `enhanceVideos` (private, tested indirectly via `getRecommended` which itself is untested).
- **Controller:** 28 endpoints. 15 tested. 13 untested (see finding #4).
- **Error paths:** Moderation rejection (create/update), block check (getById), scheduled-at guards, P2002 catch branches, and cache invalidation all lack tests.

### video-editor (ffmpeg-engine)
- **89 tests** covering command building logic across 17 describe blocks. Excellent coverage.
- **Gap:** `boomerang` parameter untested. Source code replication risk (finding #42).
- **Not tested:** `executeExport`, `cancelExport`, progress callbacks — these are native-module-dependent and cannot be tested in Jest. Acceptable limitation.

### video-replies (service + controller)
- **Service:** 4 public methods, all have at least one happy-path test. Missing: pagination hasMore=true, user enrichment, REEL comment-not-found, already-deleted reply.
- **Controller:** All 4 endpoints tested with delegation verification.

### subtitles (service + controller)
- **Service:** 4 public methods, all thoroughly tested including error paths. Best-covered module in this audit.
- **Controller:** All 4 endpoints tested.

### thumbnails (service + controller)
- **Service:** 6 public methods + 2 private. Public methods tested at happy-path level. Private `verifyContentOwnership` and `checkForWinner` lack dedicated tests. Critical: DTO case mismatch between controller (lowercase) and service (uppercase) is a latent production bug.
- **Controller:** All 5 endpoints tested.
