## Top Priority Fixes

1. **channel-posts `like`/`unlike` (C):** These are transactional operations with `$executeRaw` counter updates and junction table management. Zero coverage. Need: happy path, already-liked conflict, post-not-found, mock `channelPostLike` + `$transaction`.

2. **reports `create` urgent auto-hide (C):** When reason is NUDITY/VIOLENCE/TERRORISM, content is immediately hidden. This is a legal compliance path (CSAM, EU TCO). Zero coverage. Need: test that `post.update({isRemoved: true})` is called when reason is urgent.

3. **reports `create` mass-report abuse (C):** Rate limiting to prevent weaponized reporting. Zero coverage. Need: test that >10 reports/hour throws `BadRequestException`.

4. **reports `resolve` action effects (M x4):** CONTENT_REMOVED, PERMANENT_BAN, TEMP_MUTE, WARNING -- all resolve side-effects untested. The resolve test only verifies the transaction was called, not what happens inside it.

5. **moderation `resolveAppeal` (M):** Entire appeal resolution flow (accept/reject, content restoration, user unban) has zero coverage. This is an admin-only endpoint but a complete logical pathway.

6. **channels `getRecommended` (H+M):** Entirely untested endpoint + service method. Uses raw SQL (`$queryRaw`). Both controller delegation and service logic have zero tests.

7. **follows private-account access control (M x3):** `getFollowers`/`getFollowing` enforce privacy but the `ForbiddenException` paths for non-follower and anonymous viewers are untested.
