#### Notes on Non-Service Specs (story-stickers, publish-fields, giphy-service, create-sheet)
These 4 spec files (~1,289 lines) test **data shapes and client-side logic only** -- they never instantiate StoriesService or call any Prisma mock. They are unit tests of domain knowledge (sticker types, publish fields, GIPHY URL construction, create sheet layout) but provide zero backend coverage. They are useful for documentation/regression but inflate the test:source ratio without testing actual service behavior.
