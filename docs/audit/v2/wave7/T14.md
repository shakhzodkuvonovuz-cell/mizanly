These are NOT integration tests -- they are controller-level unit tests with mocked dependencies. True integration tests exist only in `integration-db/` which hits real PostgreSQL. |
| 10 | H | api/test | WebSocket gateway (chat.gateway.ts) | NOT TESTED | No E2E or integration test for the Socket.io gateway. ## Section 5: What IS Well-Tested (for completeness)

**API integration-db tests (real PostgreSQL):** 8 spec files covering feed visibility (16 tests), search indexing (9 tests), ban enforcement (11 tests), scheduled content (10 tests), payment webhooks (9 tests), gift atomicity (9 tests), account deletion (12 tests), follow counters (10 tests). These are genuine integration tests with real SQL execution. Total: ~86 tests against real DB.

**Go livekit-server handler tests:** ~105 tests covering all 16 endpoints' error paths, auth checks, validation, and several E2E lifecycle flows (create->delete, group decline, leave logic). Mock store is comprehensive (implements all 25 Querier methods).

**Go e2e-server handler tests:** ~30 tests covering only utility functions (validatePathParam, extractPathParam, readJSON, writeJSON, writeError, concat, hashUserID, ExtractBundleTargetID). Zero handler endpoint tests.
