# T01 — Test Coverage Audit: auth, users, two-factor, devices, settings, privacy

## Methodology
For each module: enumerated every public service method and controller endpoint, then searched all spec files for corresponding test calls. Evaluated assertion quality, error-path coverage, edge cases, and mock correctness.

## Summary by Module

### auth (9 service methods, 5 endpoints)
- **Tested:** syncClerkUser (2), deactivateByClerkId (2), checkUsername (2), getMe (4), setInterests (1), getSuggestedUsers (2), register (8). Controller: register (2), me (1), checkUsername (5), setInterests (1), suggestedUsers (1). Webhooks: user.created (1), user.updated (1), user.deleted (1), invalid sig (1), missing secret (1), missing rawBody (1).
- **Untested critical paths:** COPPA age rejection, ToS rejection, re-registration cooling period, Redis rate limiting, minor flag, trackLogin, session revocation pub/sub, webhook idempotency dedup.

### users (~30 service methods, 27 endpoints)
- **Tested:** getProfile (7), updateProfile (4), deactivate (2), deleteAccount (1), getSavedPosts (1), getSavedThreads (1), getSavedReels (1), getSavedVideos (1), getWatchLater (1), addWatchLater (1), removeWatchLater (1), getWatchHistory (1), clearWatchHistory (2), getDrafts (2), getQrCode (1), getAnalytics (1), getUserPosts (1), getUserThreads (1), getFollowers (1), getFollowing (1), report (2), findByPhoneNumbers (5), getSimilarAccounts (4), getPopularWithFriends (2), getMutualFollowers (1), getMe (3).
- **Untested critical paths:** requestAccountDeletion, cancelAccountDeletion, reactivateAccount, updateNasheedMode, getLikedPosts, getReferralCode, requestVerification, exportData delegation. 2 cron jobs untested.

### two-factor (8 service methods + 1 cron, 6 endpoints)
- **Tested:** setup (7), verify error paths (5), validate (2), disable error paths (3), getStatus (2), useBackupCode error paths (3), encryption helpers (13), key rotation cron (10), controller delegation (7).
- **Untested:** verify success path (enabling 2FA), disable success path, useBackupCode success path, validateStrict not-enabled case. All "happy path" service flows that actually mutate state are untested.

### devices (9 service methods, 5 endpoints)
- **Tested:** register (2), unregister (1), getActiveTokensForUser (2), getActiveTokensForUsers (1), getSessions (2), logoutSession (1), logoutAllOtherSessions (2), touchSession (3), cleanupStaleTokens (3), controller delegation (5).
- **Untested:** token-belongs-to-another-user deactivation path.

### settings (14 service methods, 14 endpoints)
- **Tested:** getSettings (2), updatePrivacy (6), updateNotifications (1), updateAccessibility (1), updateWellbeing (1), getBlockedKeywords (1), addBlockedKeyword (1), removeBlockedKeyword (3), getQuietMode (1), updateQuietMode (1), logScreenTime (1), setScreenTimeLimit (1), controller delegation (6).
- **Untested:** logScreenTime boundary validation, setScreenTimeLimit boundary validation, getScreenTimeStats, isQuietModeActive (4 branches), getAutoPlaySetting, updateAutoPlaySetting invalid value. 9 of 14 controller endpoints missing.

### privacy (5 service methods + 2 crons, 2 endpoints)
