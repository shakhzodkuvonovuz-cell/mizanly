| 1 | H | `polls.service.ts:189-190` | `$executeRaw` in `retractVote` uses Prisma model names `"PollOption"` and `"Poll"` instead of `@@map` table names `poll_options` and `polls`. `$executeRaw` sends raw SQL to PostgreSQL, which knows only physical table names. | 2 | H | `store.go:196-208` | `UpdateSessionStatus` has no `WHERE status IN (...)` guard. Any webhook or handler can overwrite a terminal status (`ENDED`, `MISSED`, `DECLINED`) back to `ACTIVE` or `RINGING`. E.g., a delayed `participant_joined` webhook arriving after `room_finished` would flip an `ENDED` session back to `ACTIVE` via L867. | Ghost sessions that show as ACTIVE forever; corrupted call history; E2EE key re-exposure if session is flipped back to ACTIVE after WipeE2EEKey. | S — add `AND status NOT IN ('ENDED','MISSED','DECLINED')` to each UPDATE |
| 3 | M | `store.go:246-251` | `MarkParticipantLeft` does not check `RowsAffected()`.
