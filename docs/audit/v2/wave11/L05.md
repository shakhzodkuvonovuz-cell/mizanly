## Summary

**No `@ts-ignore` or `@ts-expect-error` directives found** -- good.
**No `Record<string, any>` or `Promise<any>` found in non-test code** -- good.
**No raw `<any>` generics found** -- good.

### Severity Counts

**C: 0 | H: 2 | M: 14 | L: 13 | I: 1 | Total: 30**

### Key Patterns

1. **`as any` on message creation DTOs (#7, #8):** The most critical path in the app (sending messages) has zero type safety because the gateway passes an `as any` object. An internal `_skipRedisPublish` flag is smuggled through an untyped back-channel.

2. **Unnecessary double casts on typed fields (#19):** PostCard casts `post.topics` through `as unknown as Record<string, unknown>` six times, even though `topics` already exists on the `Post` interface. Pure noise.

3. **Dead code via wrong DTO (#16):** `videos.service.ts` checks for `scheduledAt` on `CreateVideoDto` which doesn't have that field. The condition is always false.

4. **Redundant `!` assertions (#23-25, 27-28):** Seven instances of `x!.prop` immediately after a null guard that already narrows `x`. Harmless but indicates pattern of not trusting TypeScript's control flow analysis.

5. **Prisma enum bypass (#4, #10):** `'SYSTEM' as any` used instead of the proper Prisma enum import. If the enum value doesn't exist, the DB write silently produces invalid data.
