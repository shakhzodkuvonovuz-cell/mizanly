### Critical Path

**Finding #1 is a ship-blocker.** The navigation param mismatch between `communities.tsx` (sends `communityId`) and `community-posts.tsx` (reads `channelId`) means every community tap navigates to a broken screen. This is a P0 bug.

### Recurring Patterns

1. **Hardcoded dark-mode colors in StyleSheet** (findings #8, #9, #10, #11): 26+ style properties use `colors.dark.*` or `colors.text.*` instead of theme-aware `tc.*` tokens. Light mode will be visually broken across all these screens.

2. **`fontWeight` instead of `fonts.*`** (findings #12, #13, #14, #15): 7 locations use raw `fontWeight: '600'` or `'500'` instead of the project's `fontFamily: fonts.bodySemiBold` / `fonts.bodyMedium` convention.

3. **No offline awareness** (findings #5 + missing offline checks): Only `community-posts.tsx` and `create-broadcast.tsx` use `useIsOffline()`. The remaining 8 screens allow mutations (join, create circle, subscribe, promote, delete) with no offline guard. Users will get cryptic network errors.

4. **Client-side filtering with server-side pagination** (findings #2, #19, #20, #26): Communities screen downloads all communities then filters client-side. This is incompatible with pagination -- items matching the filter may be on later pages that haven't been fetched yet.
